# zhcn-mk-pages

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容涵盖不同主题，主要用于信息展示与参考。仓库本身不针对任何特定网站或域名，是纯粹的静态页面集合。

## 目录说明

项目根目录下的每个子文件夹或文件代表一个独立的 HTML 页面或页面组。目录结构如下：

```
.
├── pages/          # 存放主要 HTML 页面
├── assets/         # 页面依赖的静态资源（CSS、JS、图片等）
└── README.md       # 本文件
```

- `pages/` 目录下的每个子文件夹对应一个独立页面，文件夹内包含该页面的 HTML 文件及相关资源。
- `assets/` 目录用于存放多个页面可能共用的公共资源。

## 页面归档说明

所有页面均以静态 HTML 形式存放，不依赖后端服务。页面内容可能包含文本、图片、样式及简单交互逻辑。归档目的包括：

- 长期保存特定主题的页面快照
- 便于通过 GitHub Pages 或其他静态托管服务直接访问
- 为研究、参考或回顾提供便利

每个页面均保留原始结构与样式，不进行额外修改。如有更新需求，请参考维护说明。

## 维护说明

- 本仓库由维护者定期检查页面可用性，但不保证所有页面永久有效或内容最新。
- 如需提交新页面，请遵循以下步骤：
  1. Fork 本仓库
  2. 在 `pages/` 下创建新文件夹，命名清晰反映页面主题
  3. 将 HTML 文件及相关资源放入对应文件夹
  4. 提交 Pull Request，简要说明页面内容与来源
- 欢迎通过 Issue 报告页面失效或提出改进建议。
